# 非交互式命令执行使用指南

## 在线采集模式（需配置SSH连接）
一键式巡检（推荐）：
```
# 自动完成采集+巡检
ascend-fd-tk set_conn_config /your_path/conn.ini auto_collect_diag
```

分步巡检（适用于分批采集场景）：
```
# 第一步：采集信息
ascend-fd-tk set_conn_config /your_path/conn.ini auto_collect（可多次执行，汇总多次采集结果）
收集完成，若完成全部收集请使用 "auto_diag" 进行巡检

# 第二步：执行巡检
ascend-fd-tk auto_diag
巡检完成
```

客户定制化巡检：
```
# 第一步：采集信息
ascend-fd-tk set_conn_config /your_path/conn.ini auto_collect

# 第二步：执行特定客户类型的巡检     
ascend-fd-tk auto_inspection <客户类型>
```

集群环境连接信息配置：
1. 工具使用时，需要连接到各个设备上进行命令查询日志采集等操作。请参考[conn.ini](../ascend_fd_tk/conn.ini)配置文件格式，配置集群连接信息
2. 若支持免密连接，可以不填写密码
3. 也支持密钥方式登录。private_key配置为私钥路径配置，可在每行单独配置，也可以在config选项框中为集群所有的环境配置
4. 其中ip支持单IP配置，也支持IP范围配置（step配置：step默认为1；确保用户名和密码相同）
5. 其中1620前台等交换设备也视为交换机，请填写到switch选项框中

## 离线分析模式（需设置日志目录）
一键式巡检（推荐）：
```
# 自动完成采集（设置Host服务器日志目录、BMC日志目录和交换机日志目录）+巡检
ascend-fd-tk set_host_dump_log /your_path/host_logs set_bmc_dump_log /your_path/bmc_logs set_switch_dump_log /your_path/switch_logs auto_collect_diag
```

分步巡检（适用于分批采集场景）：
```
# 第一步：采集信息
ascend-fd-tk set_host_dump_log /your_path/host_logs set_bmc_dump_log /your_path/bmc_logs set_switch_dump_log /your_path/switch_logs auto_collect（可多次执行，汇总多次采集结果）
收集完成，若完成全部收集请使用 "auto_diag" 进行巡检

# 第二步：执行巡检
ascend-fd-tk auto_diag
巡检完成
```

客户定制化巡检：
```
# 第一步：采集信息
ascend-fd-tk set_host_dump_log /your_path/host_logs set_bmc_dump_log /your_path/bmc_logs set_switch_dump_log /your_path/switch_logs auto_collect

# 第二步：执行特定客户类型的巡检     
ascend-fd-tk auto_inspection <客户类型>
```

支持的日志类型：

1、Host服务器日志  
- `A3device日志一键采集脚本<version>.sh`收集的日志
- `link_down_collect_<version>.sh`收集的日志
- `tool_log_collection_out_version_all_<version>.sh`收集的日志

2、BMC日志  
- BMC网页“一键收集”按钮下载的日志
- `ipmcget -d diaginfo` 命令采集的日志 

3、交换机日志  
- 使用交换机`display diagnostic-information <filename>` 命令导出的结果或者查询关键命令后复制的shell回显
- 使用交换机`collect diagnostic-information` 命令导出的zip包
 

## 查看巡检报告
巡检完成后，报告自动生成至 `~/.ascend-faultdiag-toolkit/report/` 目录：

```
# 查看报告目录
ls ~/.ascend-faultdiag-toolkit/report/

# 输出示例
diag_report_20260303_201835.csv  # 巡检分析报告
inspection_errors.csv  # 客户定制化巡检报告
```

## 清理缓存
完成巡检任务后，建议清理缓存，避免影响下次巡检：

```
ascend-fd-tk clear_cache
清理完成
```
